<decisions>
## Implementation Decisions

### Visibility filter
- **D-01:** Visibility granularity is **model family** (e.g. Opus 4.7), not each raw variant id.
- **D-02:** Storage: `localStorage`, key `hapi-visible-model-families` ŌĆö string array of stable family keys.
- **D-03:** Unconfigured default: empty/missing key = **all families visible** until user saves a filter in Settings.
- **D-04:** Settings entry: one row on Settings index ŌåÆ dedicated **`/settings/models`** sub-route (Variant A drill-down; no inline mega-section on index).
- **D-05:** Optional `hapi-visible-model-families-configured: true` when user has ever saved; distinguish "all visible" default from explicit empty selection if needed.

### New session
- **D-06:** Remove `ModelSelector` from new session; `spawnSession` omits `model` (Auto).
- **D-07:** Discovery still powers Settings and composer; discovery failure UX matches Phase 1 patterns.

### Composer picker
- **D-08:** Replace flat `HappyComposerOverlays` model list with nested picker: primary menu (Auto + enabled families) + Edit ŌåÆ Options (Thinking, Fast, Context, Effort).
- **D-09:** Auto always available in composer; not part of family filter.
- **D-10:** Runtime submit: compose valid raw id from family + options; send only `model` string (no separate effort API fields).
- **D-11:** Footer link: "Manage visible modelsŌĆ”" ŌåÆ `/settings/models`.
- **D-12:** Trigger shows family + option summary (e.g. "Opus 4.7 ┬Ę Medium ┬Ę Thinking").

### Model families
- **D-13:** Input: `CursorModelSummary[]` from `getCursorModels(machineId)`; parse label/id into family key, display name, variants.
- **D-14:** Options UI enables only combinations that exist in discovered variants for that family.
- **D-15:** Machine-scoped discovery ├Ś global visibility: effective list = `families(discovered) Ōł® visibleFamilies` (or all if filter unset).

### Non-goals
- **D-16:** No separate Hub/CLI fields for effort / `modelReasoningEffort` beyond display metadata.
- **D-17:** No static model catalog in protocol ŌĆö discovery remains runtime-owned via `agent models`.
- **D-18:** No per-machine visibility profiles in v1.

### UAT / Phase 1 polish
- **D-19:** Update Phase 1 UAT items 2ŌĆō3 when implementing (new session Auto; explicit model via in-session only).
- **D-20:** CURS-02 launch wording: model selection moves to in-session only.

### Claude's Discretion
- Exact family parser heuristics and slug algorithm.
- Component split (`ModelPickerOverlay` vs refactor `HappyComposerOverlays`).
- i18n key naming under `settings.models.*` and `composer.modelPicker.*`.

</decisions>
