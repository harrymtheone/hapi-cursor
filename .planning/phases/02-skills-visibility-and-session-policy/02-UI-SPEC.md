### A. Session policy sheet (`SkillsPolicySheet`)

**Pattern:** Mobile bottom sheet (not `FloatingOverlay`). Use Radix `Dialog` with content class override:

- `fixed inset-x-0 bottom-0 top-auto max-h-[min(85vh,640px)] rounded-t-2xl`
- Backdrop `bg-black/50`, tap outside dismisses
- `padding-bottom: env(safe-area-inset-bottom)`
- Scroll body inside sheet; header and footer sticky

**Open affordance:** Icon button in composer status/toolbar cluster (same band as model control), visible when session is active and discovery query has settled or errored. `aria-expanded` tied to open state. Optional 6px accent dot on button when any skill has `enabled` or `disabled` policy (not inherited-only).

**Header:**

- Title: â€śSession skillsâ€ť
- Close: X or â€śDoneâ€ť text button (right)
- Intro: inherited helper copy (one `text-xs` block)

**Body:**

- Search `input[type=search]` when skill count > 8 (same styling as `settings/models.tsx` search)
- Loading: spinner + â€śLoading skillsâ€¦â€ť
- Error: error copy + Retry (refetch `GET /sessions/:id/skills`)
- List: `divide-y divide-[var(--app-divider)]`, sorted `project` before `user`, then `name` localeCompare
- Each row: `SkillPolicyRow`

**Footer:**

- Text link: Browse skills catalog â†’ `/settings/skills`
- Primary text button: Reset all to inherited â†’ PATCH clears explicit policy map entries (all keys removed or set inherited server-side per API design)

**Mutation UX:** Changing tri-state fires optimistic Hub patch immediately (no Apply button). Disable controls while `controlsDisabled` or patch in flight for that row. Show subtle row opacity 0.6 during pending.

### B. Tri-state control (`SkillTriStateControl`)

Three equal segments in a horizontal pill group (`flex`, `rounded-lg`, `border border-[var(--app-border)]`, inner buttons `flex-1`):

| State | Segment label | Meaning (helper tooltip optional) |
|-------|-----------------|-----------------------------------|
| `inherited` | Inherited | No HAPI override |
| `enabled` | Enabled | Force allow for session (prompt + `$` autocomplete) |
| `disabled` | Disabled | Omit from `$` autocomplete and turn skill allow-list |

Invalid skills: control `disabled`; show invalid reason; policy cannot be changed until `valid: true`.

### C. Settings catalog (`/settings/skills`)

**Chrome:** Match `ModelsSettingsPage` header â€” back, title â€śSkillsâ€ť, **no Save button** (read-only).

**Discovery:** Use active machineâ€™s session-agnostic discovery path: prefer listing via a dedicated hub endpoint or reuse first active sessionâ€™s cwd only if that is the established plan; UI must show same `CursorModelDiscoveryStatus`-style block adapted for skills (loading / error / retry). If only session-scoped API exists in v1, show inline note â€śShows skills for active session workspaceâ€ť â€” planner may collapse to `GET /sessions/:id/skills` with last-used session id.

**Rows (`SkillsCatalogRow`):**

- Name, description (2-line clamp)
- Source pill, invocation mode when set
- Validity: checkmark hint vs error text from `invalidReason`
- `pathHint` as muted secondary (never full home path)
- **No** tri-state, **no** enforcement toggle
- Footer note: policy is per-session copy

**Settings index row (`SkillsSection`):**

- Section title: â€śSkillsâ€ť
- Row: â€śSkills catalogâ€ť
- Subtitle: count summary â€” `{{count}} skills`, `No skills`, or `Could not load` on error

### D. Composer `$skill` autocomplete

Filter in `useSkills.getSuggestions` using **effective policy**:

| Policy | In autocomplete |
|--------|-----------------|
| `disabled` | Omit |
| `enabled` | Include even if discovery would hide (override) |
| `inherited` | Include if skill appears in discovery list and `valid` |

Recent-skills ordering (`recent-skills` localStorage) applies only among allowed skills. Disabled skills never appear; selecting a disabled skill via paste is out of scope (server/prompt layer may still strip).

Autocomplete UI unchanged (`FloatingOverlay` + `Autocomplete`); only data source filtered.

## Interaction Contract

### Session policy (SKIL-02)

| Rule | Behavior |
|------|----------|
| Storage | `metadata.skillPolicy` on session; web reads via TanStack session query; writes via Hub PATCH + SSE strict patch |
| Default | Missing key â‡’ inherited |
| Resume | Sheet reopen shows persisted policy (Phase 5 INTG-01 E2E) |
| Busy session | Policy edits allowed while idle; disable when `controlsDisabled` (same as model picker) |
| Bulk reset | One tap; no confirmation; all explicit entries cleared |

### Discovery honesty (SKIL-01)

| Field | Display |
|-------|---------|

| `description` | Secondary |
| `source` | Pill |
| `invocationMode` | Text when present |
| `valid` / `invalidReason` | Error styling when false |
| `pathHint` | Muted; redacted |

| Rule | Behavior |
|------|----------|
| Default badge | â€śHAPI session policyâ€ť on every session sheet row |

| Copy | Never imply Cursor blocks skills unless badge is Cursor enforced |
| Settings catalog | Show info callout at top: â€śEnforcement labels apply in active session policy.â€ť with default HAPI explanation |
