<objective>
Close the late-ready stale-completion-marker gap (CURS-04 / CR-01, truth #4) so a delayed `turn-completed` activity arriving after `markMessageQueued()` cannot re-persist `turnCompletionMarker` or flip `thinking` to false.

Purpose: 01-13 made the durable completion marker survive a basic refetch and 01-08 wired `markMessageQueued()` to clear it for new work, but `recordSessionActivity({ kind: 'turn-completed' })` unconditionally writes the marker, sets `thinking=false`, and deletes `pendingThinkingUntilBySessionId`. A stale ready event landing inside the queued-thinking grace window therefore reverts the row to completed/unread while the user is actively waiting on new work. Verifier truth #4 fails because no test covers this ordering.

Output: `recordSessionActivity` consults `pendingThinkingUntilBySessionId` before applying the completion marker; if the session is currently `thinking` AND the pending guard is in the future, the activity is treated as stale (touches `updatedAt` only). A new bun:test regression locks the late-after-new-work behavior. No public API or schema changes.
</objective>

<interfaces>
Existing runtime facts:
- `markMessageQueued(sessionId, time)` sets `session.thinking = true`, clears `session.turnCompletionMarker`, calls `store.sessions.clearSessionTurnCompletionMarker(...)`, and writes `pendingThinkingUntilBySessionId.set(session.id, nextTime + QUEUED_MESSAGE_THINKING_GRACE_MS)` (15_000ms).
- `recordSessionActivity(sessionId, updatedAt, activity)` for `activity.kind === 'turn-completed'`:
  - calls `store.sessions.setSessionTurnCompletionMarker(sessionId, nextUpdatedAt, nextUpdatedAt)` unconditionally,
  - sets `session.thinking = false`, `session.turnCompletionMarker = nextUpdatedAt`,
  - deletes `pendingThinkingUntilBySessionId.get(session.id)`,
  - emits a session-updated patch with `statusKind: 'completed'` and `completionMarker: session.updatedAt`.
- `pendingThinkingUntilBySessionId` is the single source of truth for the queued-thinking grace window; checking it costs one Map.get with no I/O.
- `store.sessions.touchSessionUpdatedAt(sessionId, nextUpdatedAt)` is the existing non-marker write used for the `'message'` activity kind.
- All tests in `sessionLivenessService.test.ts` import from `bun:test`.
</interfaces>
