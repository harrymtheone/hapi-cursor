## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Status dot gaps, inline icon gaps, radio inner dots |
| sm | 8px | Compact row gaps, composer/status item gaps, floating overlay row rhythm |
| md | 16px | Composer horizontal padding, selector input padding, default control spacing |
| lg | 24px | New-session section padding, empty-state grouping |
| xl | 32px | Large empty-state vertical gap, overlay max comfort spacing |
| 2xl | 48px | Major empty-state and panel separation on larger screens |
| 3xl | 64px | Page-level empty space only |

Exceptions: interactive icon-only controls must keep at least a 32px visual box and a 44px effective touch target when placed in the mobile composer or session list. Status-only dots may render at 8px but must sit inside a row/control with at least 32px height.

Phase-specific layout contracts:

| Surface | Contract |
|---------|----------|
| New-session runtime selector | Keep the existing section rhythm: `px-3 py-3`, `gap-1.5`, `text-sm` control height from `py-2`. Discovery loading/error/retry content stays inside the selector section, not as a global banner. |
| Composer model info box | Place model/effort status adjacent to the existing composer status bar/model control area, above or inside the `HappyComposer` shell, never in the chat timeline. Preserve composer outer padding `px-3 pt-2 pb-*` and rounded `20px` input shell. |
| Floating model selector | Use the existing `FloatingOverlay` pattern above the composer with max height 320px. Rows use `px-3 py-2`, `gap-2`, `text-sm`, and hover/selected styles from the current permission/model overlay. |
| Session list indicators | Add a compact leading or trailing indicator inside the existing row; do not add a second text line for model/effort. Row density remains `px-2.5 py-2`, `gap-1`, title `text-sm`, path `text-xs`. |

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 12px | 600 | 1.3 |
| Heading | 20px | 600 | 1.2 |
| Display | 28px | 600 | 1.2 |

Use exactly two weights in new UI: regular 400 and semibold 600. Raw Cursor model ids are code-like values: render at 14px in the selector and composer info box, with system monospace only when the surrounding UI needs to distinguish raw ids from prose. Do not abbreviate or rename model ids; the raw id is the primary label.

Phase-specific text hierarchy:

| Surface | Contract |
|---------|----------|
| Selector label | `text-xs font-medium text-[var(--app-hint)]`; include optional/auto clarification as normal-weight helper text. |
| Model rows | Raw model id at `text-sm font-medium`; optional discovered label/capability hint at `text-xs text-[var(--app-hint)]`. |
| Composer model info | Current model id at `text-xs` or `text-sm` depending available width; runtime state label at `text-[10px]` or `text-xs`. |
| Session list status label | Visually hidden or accessible through `aria-label`/title. Visible text is allowed only for an error short hint. |

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--app-bg)` fallback `#ffffff` / dark `#1c1c1e` | Main app background, composer outer area, select background |
| Secondary (30%) | `var(--app-secondary-bg)` fallback `#f3f4f6` / dark `#2C2C2E` | Composer shell, selected session row, floating overlay hover rows, cards |
| Accent (10%) | `var(--app-link)` fallback `#111827` / dark `#ffffff` | Focus rings, selected radio dots, active model selector affordance, primary CTA |
| Destructive | `var(--app-badge-error-text)` fallback `#b91c1c` / dark `#fca5a5` | Discovery/launch/switch failure text and destructive actions only |

Accent reserved for: selected model radio indicator, model selector focus ring, retry link/button emphasis, primary "Start session" action, and proven clickable hot-switch affordance. Do not color all runtime labels with accent.

Status colors:

| State | Visual |
|-------|--------|
| Running or thinking | Spinner-style icon using `text-[var(--app-hint)]` or existing blue activity color when paired with a status label |
| Waiting for input | Single 8px yellow dot using `#FF9500` or `var(--app-badge-warning-text)` |
| Waiting for approval | Same single yellow dot as waiting for input; do not stack or prioritize separate indicators |
| Input and approval both pending | One yellow dot only |
| Error | Single 8px red dot using `var(--app-badge-error-text)`; visible short hint allowed only if the row needs disambiguation |
| Completed with unread result | Single 8px green dot using `var(--app-badge-success-text)` |
| Completed after viewed / inactive / no attention | Gray dot or no emphasized indicator using `var(--app-hint)` at reduced opacity |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Start session |
| Empty state heading | No Cursor models found |
| Empty state body | Continue with auto, or retry discovery after checking the local Cursor CLI. |
| Error state | Could not load Cursor models. Retry discovery, or continue with auto. |
| Destructive confirmation | None introduced by this phase. Existing archive/delete confirmations remain unchanged. |

Additional required copy:

| Situation | Copy Contract |
|-----------|---------------|
| Auto/default selector option | `Auto (unspecified)` in English; equivalent locale text must mean no explicit model will be passed. |
| Discovery loading | `Loading Cursor models...` |
| Discovery retry action | `Retry discovery` |
| Discovery safe reason | Show one short localized reason such as `Cursor CLI unavailable`, `Not authenticated`, `Timed out`, or `No models returned`. Never show raw stderr in the normal mobile UI. |
| Launch rejection | `Cursor rejected the selected runtime config. Choose another model or continue with auto.` |
| Read-only model box | `Model changes apply before launch only` or `Switching unavailable for this runtime`, depending actual runtime status. |
| Applies-next-run state | `Applies next run` |
| Applying state | `Applying...` |
| Applied state | `Applied` |
| Failed switch state | `Switch failed. Retry` plus a short safe reason if available. |

Use i18n keys grouped near existing `newSession.*`, `misc.*`, and composer/session keys. English and Simplified Chinese locales must be updated together when implementation adds these strings.

## Interaction Contract

### New Session Runtime Discovery

| State | Required Behavior |
|-------|-------------------|
| Panel opens | Trigger machine-scoped Cursor model discovery when the new-session panel opens and a machine is selected. Use a short cache. |
| Default | Selector value is auto/unspecified. Do not pass or persist a model unless the user explicitly selects a discovered model id. |
| Loading | Keep the selector visible with disabled control and loading copy. Do not block directory/session type controls. |
| Success | Show raw Cursor model id as the primary row text. Optional label/grouping may be secondary only. |
| Empty/error | Show safe reason, retry action, and keep `Auto (unspecified)` available. |
| Effort | Hide separate effort controls unless discovery or verified runtime behavior proves separate effort support. Do not hardcode common effort options. |
| Launch failure | If Cursor rejects selected model/effort, fail clearly. Do not silently fall back to auto or another model. |

### Composer Runtime Control

| State | Required Behavior |
|-------|-------------------|
| Location | Show chosen model/effort near the chat input/model control. This is the canonical location for runtime config visibility. |
| Hot switch unsupported or unproven | Model information box is read-only, cannot open a selector, and communicates unavailable or applies-next-run status. |
| Hot switch proven available | The model information box may open the existing floating selector pattern. It must show only discovered/runtime-approved options. |
| Busy/thinking/tools running | Disable switching. Copy should say switching is available when the session is idle. |
| Applying/applied/failed | Feedback stays inside the model information box. Do not insert model-switch events into the chat timeline. |
| Applies next run | Show a non-success neutral/warning state; do not claim the active process changed. |
| Keyboard shortcut | Existing model cycling shortcuts must not be exposed unless runtime support is true and options exist. |

### Session List Status

| State | Required Behavior |
|-------|-------------------|
| Model/effort | Do not render model or effort text in mobile session rows. This phase explicitly corrects earlier wording. |
| Compact indicator | Render one compact attention/status indicator per row. Running/thinking use spinner-style icons; waiting input/approval use one yellow dot; error uses red; unread completed uses green; viewed completed/inactive uses gray. |
| Accessibility | Every indicator needs an accessible label. Normal visible row text remains title, path, todo progress, pending count if retained, and relative time. |
| Multiple pending states | If input and approval are both pending, show one yellow dot. No stacked dots, badges, or priority ladder. |
| Read state | Green completed indicator clears to gray after the user opens/views the completed result. |

## Implementation Constraints

| Constraint | Contract |
|------------|----------|
| Stack | Use React 19, Tailwind 4, TanStack Query, assistant-ui, existing local UI primitives, and existing `useTranslation()` i18n. |
| Runtime truthfulness | No hot-switch affordance unless implementation proves true runtime support. Otherwise show read-only or applies-next-run state. |
| Wire state | Runtime display must derive from shared session fields and strict SSE patches, not web-only hidden state. |
| Mobile density | Preserve current PWA row and composer density; no desktop-only controls that require hover to understand. |
| Timeline cleanliness | Runtime switch success/failure must not create chat timeline messages. |
| Security | Discovery failure UI must use safe categorized reasons; raw stderr belongs in local logs only. |
