<objective>
Close UAT Test 5 part 2 (CURS-04 / D-15): viewed completion markers must survive a page refresh. Replace the React-only `viewedCompletionMarkers` `useState` in `web/src/components/SessionList.tsx` with a `localStorage`-backed map keyed by session id, so refreshed sessions whose completion marker was previously viewed render as gray BEFORE the user clicks anything.

Purpose: `web/src/components/SessionList.tsx` line 85 currently declares `const [viewedCompletionMarkers, setViewedCompletionMarkers] = useState<Record<string, number>>({})`. On every page mount/refresh this map starts empty; only the currently-selected session is re-marked via the effect at lines 103-111. Every other completed session re-appears as green/unread after refresh. Per Plan 01-08 SUMMARY (and the `STATE.md` decision "Keep completed-marker read state local to Web; Hub only emits completionMarker values for runtime truth"), viewed state intentionally lives in Web, not in the shared protocol ŌĆö but the team never added browser-side persistence to survive refresh. This plan adds it.

Output: A single-file production change in `SessionList.tsx` that (a) hydrates the viewed-markers map from `localStorage` on mount via a lazy `useState` initializer, (b) write-throughs to `localStorage` inside `markCompletionViewed`, (c) prunes entries whose session id is no longer in `props.sessions`, and (d) guards storage access with try/catch + `typeof window !== 'undefined'`. Plus a co-located Vitest regression suite (`SessionList.viewed-persistence.test.tsx`) covering hydrate-on-mount, write-through, prune, and try/catch resilience.
</objective>
