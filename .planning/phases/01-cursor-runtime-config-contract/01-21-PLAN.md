<objective>
Close the remaining UAT Test 6 gap: after a page refresh, all previously viewed completed sessions except the currently selected chat turn from gray back to green.

Purpose: `SessionList` currently treats every `props.sessions` change as authoritative. During refresh, `useSessions`/TanStack Query can render `sessions=[]` while loading. The prune effect in `SessionList.tsx` then saves `{}` to `localStorage`, deleting all persisted `viewedCompletionMarkers`. When real sessions load later, the selected-session effect only re-marks the current chat, so other previously viewed completed sessions render green/unread again.

Output: one targeted regression test in `SessionList.viewed-persistence.test.tsx` and the smallest production change in `SessionList.tsx`: skip persisted viewed-marker pruning while the list is in a transient empty loading state. Do not modify `useSessions.ts` or `_factory.ts` unless the test proves `SessionList` cannot reliably distinguish transient loading from authoritative empty data.
</objective>
