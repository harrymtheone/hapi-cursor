## What Was Built

Shared protocol contracts that all downstream plans (Hub persistence, ingest, API enrichment, Web reducer) can import from a single source of truth.

**`ToolCallProjectionSchema`** (D-14, SPEC req 2):
- Strict Zod schema with 4-value status enum (`pending | in_progress | completed | failed`)
- Required: `callId` (min 1), `name` (min 1), `input` (unknown), `status`, `startedAt`
- Optional: `result`, `error`, `completedAt`, `permission` (strict sub-object)
- Exported as both schema constant and `ToolCallProjection` inferred type

**`SyncEventSchema` arm** (`tool-call-projection-updated`, D-08):
- Extends `SessionChangedSchema` (sessionId) with `callId: z.string().min(1)` and `projection: ToolCallProjectionSchema`
- Additive discriminated union arm â€” no existing arms modified

- `toolCalls?: Record<string, ToolCallProjection>` â€” optional so existing Hub route builders compile unchanged
- Plans 02+ will populate this field; Web consumers treat absence as empty map
