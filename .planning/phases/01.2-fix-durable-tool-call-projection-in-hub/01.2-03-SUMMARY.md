# Phase 01.2 Plan 03: Wire Hub Ingest and Message Pagination Summary

**One-liner:** sessionHandlers now upserts durable tool-call projections and emits `tool-call-projection-updated` SSE before `message-received`; `getMessagesPage` runs one-time reconcile per session and returns `toolCalls` map scoped to the returned page's callIds.

## What Was Built

**`hub/src/socket/handlers/cli/sessionHandlers.ts`** — Ingest upsert hook (D-02, D-03, D-08):
- Import `extractToolCallEventsFromMessageContent` and `mergeToolCallProjection` from `toolCallProjection.ts`
- After `store.messages.addMessage`, extract tool-call events from message content
- For each event: fetch existing projection from `store.toolCalls`, merge with `mergeToolCallProjection`, upsert (errors caught silently — oversize/schema violations are non-critical sidecar)
- Emit `onWebappEvent({ type: 'tool-call-projection-updated', sessionId, callId, projection })` **before** existing `message-received` emit (Pitfall 3 ordering fix)
- Raw message path unchanged; `addMessage` is never skipped (D-03, D-13)

**`hub/src/sync/messageService.ts`** — Reconcile gate + page enrichment (D-04, D-06):
- `private readonly reconciledSessions = new Set<string>()` — once per session per `MessageService` instance
- At start of `getMessagesPage`: if `!reconciledSessions.has(sessionId)`, call `reconcileSessionToolCalls(sessionId, this.store)` then mark reconciled (T-01.2-03 DoS mitigation)
- After building `messages` array: call `collectCallIdsFromDecryptedMessages(messages)`, then `store.toolCalls.getBySessionAndCallIds(sessionId, [...callIds])`, return `{ messages, page, toolCalls }` — empty `{}` when no callIds
- Return type updated to include `toolCalls: Record<string, ToolCallProjection>`

**`hub/src/sync/syncEngineMessage.ts`** — passthrough return type updated to include `toolCalls: Record<string, ToolCallProjection>`.

**`hub/src/web/routes/messages.ts`** — No logic change needed; the enriched shape passes through `SyncEngine.getMessagesPage` naturally.
