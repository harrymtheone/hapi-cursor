# Phase 01.2 Plan 02: Hub SQLite Persistence and Pure Merge/Extract Logic Summary

**One-liner:** Schema v12 `tool_calls` table with FK cascade, `ToolCallStore` upsert/read facade, and pure `mergeToolCallProjection` / `extractToolCallEventsFromMessageContent` / `reconcileSessionToolCalls` functions — all test-driven and typecheck-clean.

## What Was Built

**Schema v12 `tool_calls` table (D-01, D-05):**
- `CREATE TABLE tool_calls (session_id, call_id, projection TEXT NOT NULL, updated_at INTEGER, PRIMARY KEY (session_id, call_id), FOREIGN KEY (session_id) REFERENCES sessions(id) ON DELETE CASCADE)`
- `idx_tool_calls_session` index on `session_id`
- `REQUIRED_TABLES` updated to include `tool_calls`
- `SCHEMA_VERSION` bumped from 11 → 12

**`hub/src/store/toolCalls.ts`** (upsert + read):
- `upsertToolCall(db, sessionId, callId, projection)` — JSON.stringify, 64KB cap before INSERT, ON CONFLICT upsert
- `getBySessionAndCallIds(db, sessionId, callIds[])` — `safeJsonParse` + `ToolCallProjectionSchema.safeParse` on read; corrupt/invalid rows silently skipped (T-01.2-04)

**`hub/src/store/toolCallStore.ts`** — `ToolCallStore` class facade wired as `readonly toolCalls` on `Store`.

**`hub/src/sync/toolCallProjection.ts`** (pure functions):
- `extractToolCallEventsFromMessageContent(content, messageAt)` — handles Cursor wire (`type: 'cursor'`, data.type `tool-call`/`tool-call-result`) and legacy `tool_use`/`tool_result` array blocks; never mutates input (D-03)
- `mergeToolCallProjection(prev | null, event)` — pure; result-before-start converges to correct status; never downgrades non-placeholder name to empty
- `reconcileSessionToolCalls(sessionId, store)` — scans session messages, merges per-callId, upserts via `store.toolCalls`
- `collectCallIdsFromDecryptedMessages(messages)` — collects callId set from raw `DecryptedMessage[]` for page enrichment

**`hub/README.md`** Database section — schema v12 upgrade instructions: stop, backup, delete, restart; no runtime migration policy.
