<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| process env + `~/.hapi/` files → `loadConfig()` | All untrusted bootstrap input crosses this boundary; `loadConfig` is the trust gate. |
| HTTP / Socket.IO clients → CLI bearer auth | `constantTimeEquals(presented, expected)` is the timing-attack mitigation; DI must not regress this. |
| `~/.hapi/jwt-secret.json`, `~/.hapi/owner-id.json` → JWT signing | Hub trusts disk-loaded secrets; DI changes who passes `dataDir` but not the disk gate. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-02-01 | Spoofing | `hub/src/web/routes/cli.ts`, `hub/src/web/routes/auth.ts`, `hub/src/socket/server.ts` (`constantTimeEquals`) | mitigate | DI swaps the SOURCE of `cliApiToken` from singleton to constructor parameter; the `constantTimeEquals(presented, expected)` call is preserved verbatim at all three sites. Acceptance criterion has `rg` proof that all three files still call `constantTimeEquals`. |
| T-10-02-02 | Information Disclosure | `loadConfig()` error messages | mitigate | Errors reference field/env names + settings file path; never values. `cliApiToken` value is loaded after env/field rejection so a malformed-settings error cannot include the token. |
| T-10-02-03 | Tampering | Frozen Config object | mitigate | `Object.freeze` is applied recursively (deepFreeze helper) and `corsOrigins` is copied + frozen so a downstream consumer cannot mutate `config.corsOrigins.push('http://evil.example')` after startup. Asserted by frozen-mutation tests. |
| T-10-02-04 | Spoofing | Stale `WEBAPP_*` env var → wrong listen host/port | mitigate | `rejectOldEnvVars` throws at startup naming the new env var, so users cannot accidentally bind to a default they did not intend (D-161 + Pitfall 4). |
| T-10-02-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed in this plan; package legitimacy gate not applicable. |
</threat_model>
