## Spacing Scale

Declared values (multiples of 4) — unchanged from Phase 01/02 baseline:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Pill padding (`px-1.5 py-0.5`), tag gaps |
| sm | 8px | Header icon cluster gap, row inner gaps |
| md | 16px | Sheet header/body horizontal padding (`px-6`), row `py-3` |
| lg | 24px | Sheet header vertical padding |
| xl | 32px | Empty-state vertical grouping |
| 2xl | 48px | Empty-state rhythm |
| 3xl | 64px | Page-level only |

Exceptions:

| Case | Contract |
|------|----------|
| `SessionHeader` icon buttons | `h-8 w-8` hit area (matches Files / Outline today) |
| Sheet list rows | `min-h-[48px]` (read-only; no tri-state control — shorter than Phase 02 `56px` policy rows) |
| Scope / Invalid pills | Minimum touch readability via padding; pills are not separate tap targets |

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Caption | 12px (`text-xs`) | 400 or 600 | 1.3 |
| Body | 14px (`text-sm`) | 400 or 600 | 1.4 |
| Sheet title | 16px (`text-base`) | 600 | 1.2 |

Use exactly two weights: 400 (`font-normal`) and 600 (`font-semibold`).

| Surface | Hierarchy |
|---------|-----------|
| Skill name (primary) | `text-sm font-semibold text-[var(--app-fg)]`, single-line `truncate` |
| Scope pill (Global / Local) | `text-xs font-normal` inside pill |
| Invalid pill | `text-xs font-semibold` inside pill |
| Sheet title | `text-base font-semibold` |
| Loading / empty / error | `text-sm` title + `text-xs` body per copy table |
| Done control | `text-sm font-medium text-[var(--app-link)]` |

**Not shown in session sheet:** description, `invocationMode`, `pathHint`, `invalidReason` body (D-11).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--app-bg)` | Sheet scroll body, session page background |
| Secondary (30%) | `var(--app-secondary-bg)` / `var(--app-subtle-bg)` | Header icon hover, scope pill background |
| Accent (10%) | `var(--app-link)` | Done button, Retry link, header Skills button hover text (optional active when sheet open) |
| Destructive | `var(--app-badge-error-text)` + `var(--app-badge-error-bg)` + `var(--app-badge-error-border)` | **Invalid** pill only |

Accent reserved for: sheet **Done**, discovery **Retry**, and optional `aria-expanded` active state on header Skills button when sheet is open. Do not accent skill names or scope pills.

| Pill | Background | Text | Border |
|------|------------|------|--------|
| Global / Local (scope) | `transparent` or `var(--app-subtle-bg)` | `var(--app-hint)` | `var(--app-border)` |
| Invalid | `var(--app-badge-error-bg)` | `var(--app-badge-error-text)` | `var(--app-badge-error-border)` |

Invalid skills: list row has **no** left error border strip (Phase 02 policy row border removed); invalidity communicated **only** via red Invalid pill beside scope pill (D-11).

### A. Session header — Skills entry (D-08)

**Placement:** Icon button in `SessionHeader` toolbar, **immediately after Files**, **before Outline** when `onOpenOutline` is set.

**Sizing:** Match Files / Outline — `flex h-8 w-8 items-center justify-center rounded-full`, hint color + secondary hover.

**Labels:** `title={t('session.skills.title')}` and `aria-label={t('session.skills.title')}`.

**Behavior:**

- Visible whenever session is shown with skills discovery available (`sessionId` + `api` present — same gating as today’s composer sheet minus policy callbacks).
- `aria-expanded={skillsSheetOpen}` when sheet mounted.
- Optional: when sheet open, apply `text-[var(--app-link)]` on icon (accent reserved list). **No** policy dot badge.

**Wire:** Parent passes `onOpenSkills` or manages open state + renders `SessionSkillsSheet`.

### B. Session skills sheet (D-09–D-11)

**Pattern:** Reuse Phase 02 bottom **Dialog** shell from `SkillsPolicySheet` (mobile-safe). Radix `Dialog` + `DialogContent`:

- `fixed inset-x-0 bottom-0 top-auto max-h-[min(85vh,640px)] rounded-t-2xl`
- `padding-bottom: max(1rem, env(safe-area-inset-bottom))`
- Backdrop dismiss; sticky header; scrollable body
- **No footer** region (no reset, no settings link)

**Header:**

- Title: `t('session.skills.title')`
- Right: **Done** text button → `onOpenChange(false)`
- **No** `DialogDescription` / inherited helper block

**Body:**

| State | UI |
|-------|-----|
| Loading | Centered `session.skills.loading` |
| Error | `error` + `errorBody` + **Retry** (invalidate `queryKeys.skills(sessionId)`) |
| Empty | `emptyTitle` + `emptyBody` |
| List | `divide-y divide-[var(--app-divider)]` |

**Search:** When `skills.length > 8`, sticky search input; filter **name only** (case-insensitive substring). Do not search description.

**Sort:** `project` (Local) before `user` (Global), then `name` `localeCompare` (keep `sortSkills` helper; drop policy helpers).

**Row (`SessionSkillsRow`):**

```
[ skill name (truncate) ]     [ Local | Global pill ] [ Invalid pill if !valid ]
```

| Field | Display |
|-------|---------|
| `name` | Primary label only |
| `source === 'project'` | Pill: **Local** |
| `source === 'user'` | Pill: **Global** |
| `source` unknown / other | Omit scope pill or show neither Global nor Local (prefer omit) |
| `valid === false` | Red **Invalid** pill in addition to scope pill |
| `valid === true` | No Invalid pill |
| `description`, `invocationMode`, `pathHint`, `invalidReason` | **Hidden** |

Row layout: `flex items-center justify-between gap-2 py-3 min-h-[48px]`. Pills in a `flex flex-wrap items-center gap-1.5` cluster on the right (or below name on very narrow widths if wrap needed — prefer single row).

**Data:** `useSkills(api, sessionId)` with **no** policy parameter. List includes **all** discovered skills (valid and invalid). Invalid skills are excluded from composer `/` autocomplete only (D-03, D-14).

**Mutations:** None. No optimistic state, no PATCH, no `controlsDisabled` for policy (sheet may still respect session busy only if parent passes disable for open button — not required for list).

### C. Composer `/` skill autocomplete (D-13–D-14)

| Rule | Contract |
|------|----------|
| Trigger | Composer query `startsWith('/')` routes to skill suggestions (replace all `$` paths) |
| Allowed suggestions | `skill.valid === true` from latest `useSkills` result |
| Insert text | `/skill-name` (parallel to prior `$name` keys in suggestion objects) |
| Policy filter | **None** — no `skillPolicy`, no inherited/enabled/disabled |
| Recent skills | `recent-skills` localStorage applies only among **valid** skills; store `/name` keys |

Autocomplete UI container unchanged (`FloatingOverlay` + `Autocomplete`); data source and prefix only.

### D. Settings catalog (D-15)

Unchanged surface role: read-only `/settings/skills` with richer metadata. Update any copy that mentions “session policy” toggles. Global/Local rename in settings is **optional** this phase; session sheet **must** use Global/Local.

## Interaction Contract

| Rule | Behavior |
|------|----------|
| Discovery = availability | All `valid === true` skills are allowed for agent + `/` autocomplete; no user disable |
| Session sheet | Informational only; invalid skills visible with Invalid tag |
| Policy storage | **Removed** — no `metadata.skillPolicy`, no Hub skill-policy route, no web mutations |
| Sheet open | From header only; closing does not affect discovery |
| Busy session | Header button may follow same disable rules as Files if parent sets `controlsDisabled` (optional; not spec’d as required) |
| E2E honesty | Sheet list reflects latest `GET /sessions/:id/skills` after discovery refresh on CLI turns (backend); web refetches via query invalidation on session events as today |
