## Warnings

### WR-01: Legacy Hub infer maps `.ipynb` path edits to `Edit` instead of `NotebookEdit`

**File:** `hub/src/sync/toolCallProjection.ts:120-127`  
**Issue:** `inferToolNameFromPayload` checks generic `path` / `file_path` with `old_string` / `new_string` / `edits` and returns `'Edit'` before the `notebook_path` branch. Legacy stored input from real Cursor wire often uses `path: "…/file.ipynb"` without `notebook_path` (CLI normalizer adds `notebook_path` only on new ingest). Reconcile therefore upgrades those rows to `Edit`, not `NotebookEdit`, breaking D-17 for notebook-edit carryover. `sessionModel` advanced-tool tests cover `NotebookRead` but not this `path`-only edit shape.

**Fix:** Before returning `'Edit'` for path + edit fields, treat `.ipynb` paths like shared mapping does:

```typescript
if ('old_string' in inObj || 'new_string' in inObj || Array.isArray(inObj.edits)) {
    if (path.endsWith('.ipynb')) return 'NotebookEdit'
    return 'Edit'
}
```

Alternatively call shared `resolveHapiToolName` on a synthetic `{ editToolCall: { args: inObj } }` wrapper when inferring legacy rows.

### WR-02: Hub mapping duplicates shared allowlist (D-18 drift risk)

**File:** `hub/src/sync/toolCallProjection.ts:105-157` vs `shared/src/cursor/toolCallMapping.ts`  
**Issue:** Phase research locked D-18: Hub reconcile should reuse the same mapping rules as CLI via `@hapi/protocol/cursor`, not parallel heuristics. Plan 01.3-03 extended a separate `inferToolNameFromPayload` instead of delegating to `resolveHapiToolName` / shared helpers. CLI now infers `NotebookEdit` from `editToolCall` + `.ipynb` suffix; Hub infer does not — behavior will diverge on future allowlist changes.

**Fix:** Export a shared `inferToolNameFromStoredInput(input, output?)` (or reuse `resolveHapiToolName` + normalizer hints) from `@hapi/protocol/cursor` and call it from `resolveProjectionName`. Keep Hub-specific ordering only where stored wire lacks native `*ToolCall` envelopes.

## Info

### IN-01: Capture redaction is Linux-home-centric

**File:** `cli/src/cursor/utils/cursorNdjsonCapture.ts:15-18`  
**Issue:** `PATH_PATTERNS` redacts `/home/…` and `~/.hapi` but not macOS `/Users/…` or arbitrary absolute paths. Acceptable for dev-only capture on the project's primary Linux target, but captures on other OS layouts may retain usernames/paths in `~/.hapi/cursor-ndjson-capture/`.

**Fix:** Add `/Users/[^\s"'\\]+/g` (or platform-detect home prefix) when cross-platform capture is required.

### IN-02: `normalizeTaskArgs` skips object-shaped `subagentType` on captured wire

**File:** `cli/src/cursor/utils/normalizeToolArgs/normalizeTaskArgs.ts:15-18`  
**Issue:** Real capture fixture uses `subagentType: { unspecified: {} }` (object). Normalizer only copies string `subagentType` / `subagent_type`. Task ToolCard title/subtitle use `description` / `prompt`, so display is unaffected today; stored input retains non-canonical object shape for any consumer expecting `subagent_type` string.

**Fix:** If needed, map known object variants to a string label (e.g. first key) or omit the field explicitly in normalized output.

### IN-03: `reconcileSessionToolCalls` scans at most 200 messages

**File:** `hub/src/sync/toolCallProjection.ts:237`  
**Issue:** Reconcile loads `getMessages(sessionId, 200)`. Tool calls referenced only in older messages are never re-projected. Pre-existing cap, not introduced in 01.3, but limits D-17 backfill on very long sessions.

**Fix:** Paginate message scan or reconcile from `tool_calls` sidecar keys when full-session heal is required.
