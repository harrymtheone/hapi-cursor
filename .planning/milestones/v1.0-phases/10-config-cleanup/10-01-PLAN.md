## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| process argv → command resolver | Untrusted user input enters at `getCliArgs()` and is dispatched by `resolveCommand`. |
| process env → CLI hub launcher | The CLI sets process env before importing the hub; mismatched names cause silent misroute. |
| operator → SQLite file on disk | Hub trusts the schema version PRAGMA in the local DB. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01-01 | Spoofing | `cli/src/commands/registry.ts::resolveCommand` | mitigate | Reject `hapi server` with a typed error message naming the canonical `hapi hub`; users cannot silently invoke a stale alias that may resolve to the default cursor command. (D-160) |
| T-10-01-02 | Tampering | `cli/src/commands/hub.ts` env writes | mitigate | Rename `WEBAPP_HOST/PORT` writes to `HAPI_LISTEN_HOST/PORT`; the hub binds to the user-specified host/port instead of a silent default that could expose an unintended interface. (Pitfall 4) |
| T-10-01-03 | Information Disclosure | Store schema-mismatch error | accept | Error message includes DB path + version numbers (no secrets); D-173 explicitly requires this telemetry. |
| T-10-01-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed in this plan; package legitimacy gate not applicable. |
</threat_model>
